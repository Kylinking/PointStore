店面信息

1 创建分店

URL METHOD PARAMS RESPONSE TYPE RESPONSE
/api/v1/shops POST Name:String,
Address:String,
Phone:Integer
Status:Integer,
ParentShopID:Integer		 Object {ShopID,Name,
Address,Status,
Phone,ParentShopID,
Type:Integer}
  • 操作权限：Superman，总店，分店
  • 请求参数：
    • Name,Address,Phone 为必输字段
    • 操作ID只能是总店或Superman，操作ID是分店，则报错。
    • ParentShopID 表示店面归属。
      • 若操作ID是Superman, ParentShopID 输入 Superman ID，则创建的是总店。输入总店ID，将创建该总店下的分店，输入分店ID或无对应分店信息存在，将报错。
      • 若操作ID是总店，将忽略ParentShopID字段，创建该总店下的分店
    • Status 表示店面状态 1 为正常 0 为注销
  • 返回参数：
    • Type 表示类型 0为Superman 1为总店 2为分店 （后续可能扩展 3客户）
  • 新建分店成功后，会自动创建分店账户记录，各项数值初始为0
  • 回到顶部

2 获取店面信息

URL METHOD PARAMS RESPONSE TYPE RESPONSE
/api/v1/shops GET ShopID,Type,
Phone,Page,Size		 Array [{ShopID,Name,
Phone,Address,
Status,Type,
ParentShopID，ParentShopInfo:Object
Pages:Integer,Size:Integer}]
  • 操作权限：Superman，总店，分店
  • 请求参数
    • 全部为可选参数
    • Type 字段只有在操作ID是Superman时会起作用
      • 1：表示取总店信息，如果有总店ShopID，则只返回该总店信息
      • 2：表示取分店信息，如果有总店ShopID，则返回该总店下所有分店信息
    • Page 表示取第几页，默认为 1
    • Size 表示每页最多记录数量，默认为 20
    • ShopID 表示按ShopID查询
    • Phone 支持子串匹配
  • 响应参数
    • ParentShopID 表示该店面的归属，分店归属为总店，总店归属为Superman，Superman该字段为空
    • ParentShopInfo 为归属店的信息对象，对象内容包含响应参数的所有字段
    • Size 表示每页最多记录数量
    • Pages 表示在Size下，总量数据的页数
  • 回到顶部

  • 请求参数：

店面账户信息

1 获取店面账户信息

URL METHOD PARAMS RESPONSE TYPE RESPONSE
/api/v1/shoppoints GET ShopID,Page,Size Array [{id,CustomedPoints,
RecommendPoints,
ChargedPoints,
ShopBounusPoints,
ShopID,ShopInfo}]
  • 操作权限 Superman,分店，总店
  • 请求参数
    • 全部为可选参数
    • 当操作ID是Superman时：
      • 若ShopID是总店，则返回该总店下所有分店的账户信息
      • 若ShopID是分店，则返回该分店账户信息
      • 若ShopID为空，则返回所有分店账户信息
    • 当操作ID是总店时：
      • 若ShopID为空，或是该总店，则返回总店下所有分店的账户信息
      • 若ShopID为该总店下属分店，则返回该分店账户信息
      • 其它情况返回错误信息
    • 当操作ID是分店时：
      • ShopID若不为空，则在ShopID与操作ID相同时，返回该分店账户信息
      • 其它情况返回错误信息
  • 返回参数
    • CustomedPoints:Integer, 表示分店客户当前累积消费积分数
    • RecommendPoints:Integer,表示分店客户当前累积推荐奖励积分数，包含一次推荐与二次推荐
    • ChargedPoints:Integer,表示分店客户当前累积充值积分数
    • ShopBounusPoints:Inteter,表示分店当前累积赠送积分数
    • ShopInfo 表示分店信息
  • 回到顶部

客户账户

1 获取客户账户信息

URL METHOD PARAMS RESPONSE TYPE RESPONSE
/api/v1/userpoints GET ShopID,Phone,Page,Size Array [{id,CustomedPoints,
RecommendPoints,
ChargedPoints,
ShopBounusPoints,
RemainPoints,
IndirectRecommendPoints,
CustomerID,CustomerInfo}]
  • 操作权限：Superman，总店，分店
  • 请求参数
    • 全部为可选参数
    • 当操作ID是Superman时：
      • 若ShopID是总店，则返回该总店下所有分店的客户的账户信息
      • 若ShopID是分店，则返回该分店下的客户账户信息
      • 若ShopID为空，则返回全部分店的客户账户信息
    • 当操作ID是总店时：
      • 若ShopID为空，或是该总店，则返回总店下所有分店的客户账户信息
      • 若ShopID为该总店下属分店，则返回该分店的客户账户信息
      • 其它情况返回错误信息
    • 当操作ID是分店时：
      • ShopID若不为空，则在ShopID与操作ID相同时，返回该分店客户账户信息
      • 其它情况返回错误信息
    • Phone 根据电话号码查询指定客户账户信息
  • 响应参数
    • id 序号
    • RemainPoints:Integer 客户可消费余额
    • ShopBounusPoints 客户受赠积分
    • ChargedPoints 客户累积充值积分
    • RecommendPoints 客户累积推荐奖励积分
    • IndirectRecommendPoints:Integer 客户累积二级推荐奖励积分
    • CustomedPoints:Integer 客户累积消费积分
    • CustomerInfo: 客户信息，其中包含ShopInfo字段表示客户归属的分店信息
  • 回到顶部
